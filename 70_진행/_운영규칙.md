# ⚙️ 진행 레이어 운영규칙

> 이야기를 굴리며 **배경(canon)을 오염시키지 않고** 변화를 기록하는 법. → [[현재_정세]] · [[연대기]] · [[세계관_허브]]
>
> *(소설 전환(2026-07-11) 후: 집필의 헌법은 [[_집필_규칙]], 이 문서는 "기록·커밋 절차" 담당. [[_서술_연출_규칙]]은 세계 시뮬용으로 격하.)*

## 대원칙: canon ↔ state 분리
- **배경(00~60 폴더)** = 동결된 재료. 진행 중 *건드리지 않는다*. (예외: 명백한 모순 수정, 또는 스토리가 새 설정을 *실제로* 요구할 때 — [[세계관_범위와_경계]]의 canon 해동 절차)
- **state** = 굴러가며 변하는 것: 세계 상태는 `70_진행`, 소설 설계·원고는 `80_집필`·`90_원고`. 여기만 갱신한다.

## 갱신 절차 (회차 확정 시마다 네모가 수행)
회차가 **퇴고 이상**으로 확정되면 ([[_집필_규칙]] §7):
1. 작중 확정 사건 → **[[연대기]]에 기록**(작중 날짜·무슨 일·왜·몇 화).
2. **[[현재_정세]] 덮어쓰기** — 변한 상태(정국·인물·전쟁·경제)만.
3. **[[작중_타임라인]]·스레드 커버리지 갱신.**
4. **배경 모순 점검** — 새 전개가 canon과 안 부딪치는지. 부딪치면 보고.
5. **git 커밋** — 한 회차(또는 한 진행) = 한 커밋. 메시지에 요지.
- 균형지수 B 재계산은 **큰 국면 변동 때만** — 매 회차 강제 아님 (시뮬은 보조 도구, 2026-07-11 결정).

## 연대기 작성 형식
- 파일명: `연대기/NNN_YYYYMM_제목.md` (순번 + 인게임 시점). 예: `001_191504_봉천_사변_보고.md`
- 본문: 무슨 일 / 원인 / 결과(어느 레버·상태 변동) / 관련 배경 노트(예: [[세계관_개요]]) 링크.
- **append-only** — 지난 기록은 고치지 않는다(역사니까). 정정은 새 항목으로.

## 시뮬레이션으로 쓰기 (보조 도구 — 흐름 가늠용)
- 플롯을 짜다 "이 사건이 터지면?" 싶을 때 → 균형모델로 B 변화를 따져 그럴듯한 파장을 가늠 → [[플롯_아웃라인]] 설계에 참고.
- B가 회랑(40~60)을 벗어나면 '국면 전환'이니, 그 자체가 큰 드라마 분기 — 쓰기 전에 보고 대상.
- **중요도는 일부러 낮춰뒀다**: 시뮬이 플롯을 결정하지 않는다. 결정은 [[플롯_아웃라인]](너와 네모의 설계)이 한다.

## git
- 이 Vault는 git으로 진행 히스토리를 남긴다. `git log`로 세계의 변천을, `git diff`로 무엇이 바뀌었는지 추적·롤백 가능.
- 배경 동결 시점이 최초 커밋(baseline).
