# ⚙️ 진행 레이어 운영규칙

> 이야기를 굴리며 **배경(canon)을 오염시키지 않고** 변화를 기록하는 법. → [[현재_정세]] · [[연대기]] · [[세계관_허브]]
>
> *(짝 문서: 어떻게 서술/연출/사건을 굴릴지는 [[_서술_연출_규칙]]. 이 문서는 "기록·커밋 절차" 담당.)*

## 대원칙: canon ↔ state 분리
- **배경(00~60 폴더)** = 동결된 재료. 진행 중 *건드리지 않는다*. (예외: 명백한 모순 수정, 또는 스토리가 새 설정을 *실제로* 요구할 때 — [[세계관_범위와_경계]] 규칙)
- **진행(70_진행)** = 굴러가며 변하는 것. 여기만 갱신한다.

## 갱신 절차 (네모가 매번 수행)
1. 사건/선택 발생 → **[[연대기]]에 한 줄 기록**(날짜·무슨 일·왜).
2. **[[현재_정세]] 덮어쓰기** — 변한 상태(정국·인물·전쟁·경제)만.
3. 큰 변동이면 **[[교착_균형_모델]]로 균형지수 B 재계산** — 어느 레버를 ±얼마 흔들었는지.
4. **배경 모순 점검** — 새 전개가 canon과 안 부딪치는지. 부딪치면 보고.
5. **git 커밋** — 한 번의 진행 = 한 커밋. 메시지에 날짜·요지.

## 연대기 작성 형식
- 파일명: `연대기/NNN_YYYYMM_제목.md` (순번 + 인게임 시점). 예: `001_191504_봉천_사변_보고.md`
- 본문: 무슨 일 / 원인 / 결과(어느 레버·상태 변동) / 관련 배경 노트(예: [[세계관_개요]]) 링크.
- **append-only** — 지난 기록은 고치지 않는다(역사니까). 정정은 새 항목으로.

## 시뮬레이션으로 쓰기
- "이 사건이 터지면?" → 균형모델로 B 변화를 먼저 따져 그럴듯한 파장을 예측 → 연대기·정세에 반영.
- B가 회랑(40~60)을 벗어나면 '국면 전환'이니, 그 자체가 큰 드라마 분기.

## git
- 이 Vault는 git으로 진행 히스토리를 남긴다. `git log`로 세계의 변천을, `git diff`로 무엇이 바뀌었는지 추적·롤백 가능.
- 배경 동결 시점이 최초 커밋(baseline).
