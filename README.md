# 🌍 설정제작기 (Worldbuilding Setting Creator)

스토리 세계관을 씨앗 한 줄에서 구체화된 설정 문서로 키우는 Obsidian 기반 제작기.
**너가 씨앗을 던지면 → 네모(AI)가 살을 붙여 구체화하고, 위키링크로 엮고, 일관성을 검증한다.**

## 시작하기 (Obsidian 연결)
1. Obsidian 앱 실행 → **"Open folder as vault"** → 이 `character_test` 폴더 선택.
2. 끝. 그래프 뷰 / 위키링크 / 템플릿이 이미 켜져 있다 (`.obsidian/` 동봉).
3. 첫 화면으로 [[세계관_허브]]를 열면 모든 카테고리로 이동 가능.

> 참고: 네모(Claude Code)와 Obsidian은 **같은 폴더를 공유**할 뿐 실시간 연결은 아니다.
> 네모가 md를 고치면 Obsidian에 즉시 반영되고, 반대도 마찬가지.

## 폴더 구조
| 폴더 | 내용 |
|---|---|
| `00_세계관개요` | 전제·테마·톤 (하향식 시작점) |
| `40_체계` | 마법/기술/규칙 — 일관성의 핵심 |
| `20_지역` | 장소·지리 |
| `30_세력` | 조직·집단 |
| `50_연표사건` | 역사·사건 |
| `10_인물` | 캐릭터 |
| `60_용어개념` | 용어집 |
| `_MOC` | 인덱스(허브·카테고리별) |
| `_템플릿` | 카테고리별 양식 7종 |

## status — 설정의 성장 단계
모든 노트는 프론트매터에 `status`를 가진다:

`씨앗` → `초안` → `구체화` → `완성`

- **씨앗**: 한 줄 아이디어
- **초안**: 핵심 항목 채워짐
- **구체화**: 관계·디테일·링크 붙음
- **완성**: 모순/고아 링크 없음 검증됨

## 권장 플러그인 (선택)
- **Dataview** — MOC의 자동 목록/진행현황 표가 동작 (없어도 코드블록으로 보임)
- **Templater** — 템플릿 자동 삽입 강화 (코어 Templates로도 충분)

## 작업 방식
새 설정을 만들 땐 [[_네모_워크플로]]를 따른다. 하향식(세계→씨앗) + 핵심만 묻고 채우기.
