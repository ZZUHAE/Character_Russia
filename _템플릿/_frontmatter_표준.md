# frontmatter 표준 (v1)

> 이 Vault를 **메타데이터 기반 LLM 위키**로 운용하기 위한 계약.
> LLM은 `description`만 스캔해 노트를 열지 말지 판단하고, `moc`로 소속을 집계하며,
> `status`로 성숙도를 안다. 이 필드들이 깨지면 쿼리·선별 로드가 무너지므로 **스키마는 고정**이고
> `_scripts/frontmatter_lint.py`로 강제한다.

## 1. 내용노트 필수 필드 (8)

`00_세계관개요 · 10_인물 · 20_지역 · 30_세력 · 40_체계 · 50_연표사건 · 60_용어개념`
아래 폴더의 모든 `.md`(단, `_XXX_MOC.md` 제외)는 다음 8개를 **전부** 가진다.

| 필드 | 형식 | 규칙 |
|---|---|---|
| `type` | enum | `인물 / 지역 / 세력 / 체계 / 사건 / 용어 / 세계관개요 / 수치모델` 중 하나. (MOC 파일은 `MOC`) |
| `status` | enum | `씨앗 / 초안 / 구체화 / 완성` 중 하나 |
| `description` | 문자열(따옴표) | **한국어 1~2문장, LLM용 관련성 요약.** §3 규칙 |
| `tags` | 배열 | `[a, b, c]` |
| `created` | `YYYY-MM-DD` | 생성일. 변경 금지 |
| `modified` | `YYYY-MM-DD` | 최종 수정일. 노트 **내용**을 고칠 때마다 갱신 |
| `aliases` | 배열 | 별칭(원어명 포함 가능) |
| `moc` | 위키링크(따옴표) | 소속 MOC. §4 폴더 결정표 |

노트별 선택 필드(`position` `born` `when` `family` …)는 필수 뒤에 자유롭게 유지한다.

## 2. 필드 순서 (권장)

```yaml
type → status → description → tags → created → modified → aliases → moc → (노트별 필드)
```
기존 노트에 소급 적용 시: 기존 필드는 그대로 두고 `description`은 `status` 뒤, `modified`는 `created` 뒤, `moc`는 `aliases` 뒤에 **삽입**한다(전면 재정렬 불필요).

## 3. `description` 작성 규칙 (핵심)

- **한국어 1~2문장.** 본문 `## 개요`(있으면)를 압축해서 쓴다.
- **구체적·명사 중심.** "이 인물에 대한 노트" 같은 **제네릭 금지**. 누가/무엇을/왜 중요한지가 담겨야 한다.
- **위키링크 대괄호 제거.** `[[마리아_돌마토바]]` → `마리아 돌마토바` (본문 아님, 순수 산문).
- **반드시 큰따옴표로 래핑.** `:` `#` `[` `]` 등 특수문자가 YAML 파서를 깨뜨리는 걸 막는다.
- 내부 큰따옴표는 피하고(다시 쓰기) 어쩔 수 없으면 `\"` 이스케이프.

예)
```yaml
description: "페테르부르크 명문 돌마토프 가문의 거상 출신 개혁가. 중공업·군수 생산력을 지렛대로 정계에 올라 1915년 제국 내각 부총리(실권은 그 이상). 개혁파의 동력이자 황실파의 최대 정적."
```

## 4. `moc` 폴더 결정표

| 폴더 | `moc` 값 |
|---|---|
| `00_세계관개요` | `"[[_개요_MOC]]"` |
| `10_인물` | `"[[_인물_MOC]]"` |
| `20_지역` | `"[[_지역_MOC]]"` |
| `30_세력` | `"[[_세력_MOC]]"` |
| `40_체계` | `"[[_체계_MOC]]"` |
| `50_연표사건` | `"[[_연표_MOC]]"` |
| `60_용어개념` | `"[[_용어_MOC]]"` |

MOC 파일 자신(`_XXX_MOC.md`)과 허브(`세계관_허브`)는 `moc`를 갖지 않는다(그 자체가 색인). 단 `description`·`modified`는 갖는다.

## 5. MOC/허브 파일 필드

`_XXX_MOC.md`, `_MOC/세계관_허브.md` 등 구조 노트는 최소:
`type` · `tags` · `description`(이 색인이 무엇을 모으는지) · `modified`.

## 6. 검증

`_scripts/frontmatter_lint.py` 가 전수 검사:
필수 8필드 존재 / `type`·`status` enum / `description` 비어있지 않음·따옴표 / `moc` 값이 실재 MOC 파일로 해결 / YAML 위키링크 따옴표 여부.
→ 노트 추가·수정 후 이 스크립트로 **고아·스키마 위반 0** 을 확인한다.
